Uso de Switch

Autor: Samuel Vanegas Bueno

30 Junio 2023

La declaracion switch es una declaracion que nos permite evaluar una expresion comparando esa expresion con una instancia llamada case, ejecutando el codigo que se encuentra dentro de esta instancia, si la primera no es cierta revisara las demas, funcionando muy similar a un if.

Con esto podemos comprender que la declaracion switch tiene un parecido con la sentencia if, ademas nos permite que si queremos hacer un if con varios else, si lo hacemos con la sentencia switch nos permite ahorrar mucho codigo, por lo cual es bastante recomendable que al tener un codigo que contenga bastantes else lo hagamos con switch.

Sintaxis

switch(i) { 
  case i == 1: 
   alert('Hello World') 
   break 
  case i == 2: 
   alert('Hello') 
   break  
  default: 
   alert('Bye') 
   break 
 }

En el codigo anterior podemos ver varias cosas, para comenzar vemos la declaracion switch y al lado tenemos un booleano true, este booleano se coloca cuando las comparaciones con mas complejas, por ejemplo cuando se utilizan operadores logicos como && o ||, si no son comparaciones complejas se puede utilizar lo que se esta comparando, es este caso es la variable i, por lo que quedaria asi:

switch(i) { 
  case i == 1: 
   alert('Hello World') 
   break 
  case i == 2: 
   alert('Hello') 
   break  
  default: 
   alert('Bye') 
   break 
 }

Seguido de esto podemos encontras la instancia case, donde debemos escribir la comparacion, en el codigo se puede observar que pregunta si la variable i es igual a 1, si esta comparacion es verdadera se ejecutara el alert('Hello World'),si la sentencia no es verdadera leera la siguiente, donde nos pregunta si la variable i es igual a 2, si la sentencia es verdadera, ejecutara el alert('Hello'), si esta sentencia tampoco es verdadera ejecutara lo que se encuentra dentro del default el cual es como el else de la sentencia if, asi que si ninguno de los case anteriores es correcto siempre se ejecutara el default, este es opcional y no es obligatorio aÃ±adirlo.

Tambien podemos observar que despues del codigo que queremos que se ejecute cuando una de las sentencias case es verdadera hay un break, esto lo que indica es que ahi termina el codigo que queremos que se ejecute, por ejemplo en el primer case, vemos que despues del codigo hay un brak, lo cual dice que despues del alert('Hello World') no hay mas codigo que ejecutar, aÃ±adir el break es necesario para la correcta ejecucion del codigo.
