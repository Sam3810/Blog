Introducción a Git

Autor: Samuel Vanegas Bueno

25 Junio 2023

Git es un controlador de versiones creado por Linus Torvald, pensando en la eficiencia, confiabilidad y compatibilidad del mantenimiento de versiones de aplicaciones cuando estas tienen una gran cantidad de código fuente.

Hoy en dia Git es una de las herramientas mas utilizadas en el mundo del trabajo tech, ayudando a los desarrolladores a ser mas eficientes en su trabajo y tener un mejor manejo de sus proyectos. Muchas personas que están comenzando aun no saben utilizar Git, por lo cual si eres uno de ellos, te traigo una explicación breve de Git, algunos de sus comandos, como conectarlo con tu repositorio de GitHub y subirlo sin necesidad de entrar a la pagina.

Instalar Git

Lo primero que debes hacer es intalar Git, instalarlo es muy facil, lo unico que debes hacer es irte a esta pagina y ejecutarlo, una vez lo tengas instalado debes abrir cmd con Win + r y ahi vas a escribir cmd una vez los abras debes escribir git --version, si te regresa la version lo has instalado correctamente, si te salta un error o no te regresa nada, la instalacion fallo, asi que asegurate de que fue lo que hiciste mal, o pregunta en Stack Overflow, ahi te podran ayudar.

¿Como empezar?

Ya que te aseguraste de tener Git instalado correctamente, ahora debes ingresar lo siguiente, en cmd escribe lo siguiente: 

git config --global user.name "Tu nombre aqui"
Esto lo que hara sera registrar tu nombre, ahora escribe lo siguiente: 
git config user.email "Tu email aqui"
Esto lo que hara sera registrar tu email.

Crea tu repositorio local

Un repositorio local es donde guardaras los commits de tu proyecto para luego enviarlos a tu repositorio de GitHub, para poder crearlo debes hacer lo siguiente, dirigete a la carpeta de tu proyecto, haz click derecho sobre tu carpeta y presiona en Git Bash Here

Hacer esto abrira una especie de cmd con colores, solo que es mas facil manejar Git desde aqui que desde cmd, ahora escribe git init, esto creara una carpeta como la que se ve en la captura de pantalla, si no la ves, es que no tienes activada la opcion ver carpetas invisibles, si no sabes activarla, aqui te explican como.

La carpeta llamada .git es como el repositorio local del que te hable hace un rato, ahi se almacenaran los commits que hagas para que luego puedas enviarlos a tu repositorio de GitHub

Conectando Git a tu repositorio de GitHub

Ahora viene la parte divertida, conectar Git a tu repositorio de GitHub, hacerlo es muy simple, lo unico que debes hacer es abrir Git Bash como te lo explique anteriormente, y escribir lo siguiente: 

git remote add origin url
En la parte que dice url, debes pegar la url que se encuentra en esta parte de tu repositorio de GitHub:

Tu primer commit en Git

Ya que tengas todo esto listo vas a hacer tu primer commit, pero antes de esto debes asegurarte de entender comandos basicos como los siguientes:

git status

Este comando te mostrara el estado de tu proyecto y si tienes cambios sin añadir a tu repositorio local (.git)

git pull

Este comando es bastante util ya que te permite tomar todo el proyecto que tienes en tu repositorio de GitHub y traerlo a tu carpeta, ademas soluciona el error de que tu repositorio local está desactualizada en comparación con el repositorio remoto de GitHub.

git add

Es necesario para hacer un commit, lo que hace es añadir los cambios a tu repositorio local para luego poder hacer el commit

git commit -m ""

Hace el commit en tu repositorio local, lo que permite que puedas hacer un git push para enviarlo a tu repositorio de GitHub

git push

Toma el commit que tienes guardado en tu reposito y lo envia a tu repositorio de GitHub
